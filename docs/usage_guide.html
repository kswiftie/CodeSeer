Usage Guide

You can find the usage examples in the /examples directory (currently only tokenization_inspections_usage.py is complete).

1. Initial Setup

To begin, import the following classes from Codeseer:

from codeseer import RepoHandler, ReportCompiler

Next, initialize the instances:

  • RepoHandler: Pass your GitHub token to the constructor.
  • ReportCompiler: Pass the earlier created instance of RepoHandler to the constructor.

2. Generating a Report

Now you can call the make_report method from the ReportCompiler instance:

  • Provide the path where the report will be saved (including the filename).
  • Pass a list of checks as a dictionary, where the key is the class name and the value is the list of checks for that class.
  • Pass a list of GitHub links or code snippets for which the checks should be performed. You can also pass a tuple with the first element as the link/code and the second as a custom name for the item. Alternatively, you can let Codeseer generate the names automatically.

3. Available Checks

The following checks are available at the moment:

  • TokenizationInspections:
    • compare_files_standart
    • compare_folders_standart
    • compare_files_nn
    • compare_folders_nn
    • compare_files_with_folders_standart
    • compare_files_with_folders_nn
Note: For detailed information about the checks, refer to the documentation in /docs.
