Usage Guide

You can find the usage examples in the /examples directory.

1. Initial Setup

To begin, import the following classes from Codeseer:

from codeseer import RepoHandler, ReportCompiler

Next, initialize the instances:

  • RepoHandler: Pass your GitHub token to the constructor.
  • ReportCompiler: Pass the earlier created instance of RepoHandler to the constructor.

2. Generating a Report

Now you can call the make_report method from the ReportCompiler instance:

  • Provide the path where the report will be saved (including the filename).
  • Pass a list of checks as a dictionary, where the key is the class name and the value is the list of checks for that class.
  • Pass a list of GitHub links or code snippets for which the checks should be performed. You can also pass a pair (tuple) with the first element as the link/code and the second as a custom name for the item. Alternatively, you can let CodeSeer generate the names automatically.

3. Types of Verification

The system supports different types of verification depending on the input:

  • compare_folders — compares the contents of two directories, analyzing all relevant files inside, each with each.
  • compare_files — compares the transferred files or code snippets.
  • compare_files_with_folders — compares each of the list of files with the contents of each folder.

4. Providing Input Data

Now, let's go into more detail on how to pass input data. After specifying the check classes and the types of checks, you need to provide the input data to which the checks will be applied. Here's how it is done for each type of check:

  • compare_folders
    You need to provide links to folders on GitHub (entire repositories are also allowed).
  • compare_files
    You can pass data in several ways:
    1. Provide a list of GitHub links to individual files.
    2. Provide a list of code snippets as plain text strings.
    You can combine both types of input. Additionally, for each link or code snippet, you can specify a name that will be displayed in the report. To do so, pass the data as a pair: (link_or_code, name). Otherwise, CodeSeer will generate names automatically.
  • compare_files_with_folders
    You must pass two lists:
    • The first list contains inputs formatted like for compare_files.
    • The second list contains folder links formatted like for compare_folders.
Note: For detailed information about the verifications, refer to the documentation in /docs/inspections.
